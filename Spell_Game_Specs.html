What is The Spelling Game?

I developed The Spelling Game between February 20th and February 23rd, 2020, to fulfill a homework requirement for CSCI 4208 at the University of New Orleans.

The game is very simple to use. The user clicks a button which provides a recording of a hard-to-spell word. Then, the game provides a text box in which the user can spell the word. The game checks to determine if the word has been spelled correctly. If so, the game responds with Correct! Otherwise the game responds with Not Correct!

This is a very casual game. There are no time limits, and the user can attempt as many spellings as he or she desires.

Spelling Game Roles

System Administrator – The System Administrator has no role. The typical duties of an Administrator (OS updates, security, establishment of accounts, software installation) do not apply to the conversion of The Spelling Game to Electron format.

Regular Users and Guests – The Spelling Game will not differentiate between regular users and guests. The user flow for regular users and guests will be as described in the slide, User Story – Detailed Version.

User Story – Detailed Version

The Electron version of The Spelling Game will be distributed in a zip file. After the file is unzipped, the user will find an exe file. Clicking on the exe file will open The Spelling Game on the desktop. After reading the instructions, the user will click a button to hear the hard-to-spell word. One button is provided for each of the 26 letters of the alphabet.

A text box is provided for the user to attempt to spell the word. After the user spells the word, the game will inform the user whether he or she spelled it correctly or not. The user may attempt to re-spell as many times as he or she likes.

Alternatively, the user may continue down the list to attempt to spell other words. There is no timer or ticking clock which specifies the amount of time the user can spend on any one word.

Features Rendered by the Frontend


  • All text, whether paragraphs, "h1" elements, or anything else.
  • All 26 play buttons, one for each letter of the alphabet.
  • All 26 “Check my spelling” buttons.
  • All 26 text boxes, which are later evaluated to determine whether the user spelled the word correctly
  • All 26 "div" elements, which report whether the word has been spelled correctly or not
  • The single "a" element, which links to my source of hard-to-spell words

Backend Features Performed by Node.js

On my own, I was not aware how Electron uses Node.js. However, this website elaborates on it. Quoting from the website:

“Electron uses Node.js along with Chromium, but rather than combining the event loops of the two pieces of software together, it combines them through Node.js’ node bindings feature. In this way, the Chromium and Node.js components can be updated easily without the need for custom modification of the source code, nor compilation afterwards.”
