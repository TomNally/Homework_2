How I Built the Spelling Game

by Tom Nally


Starting out...

As part of our Homework 2 assignment, we were asked (among other things) to customize our portfolio page by making a portion of it interactive using JavaScript. Alternatively, we were given the option of developing a client-side JavaScript app. I decided that I would write a game of some kind.

I decided that a spelling game would be doable, and would require no animation. But what form would it take? I considered a fill-in-the-blank spelling game, where a few letters would be provided, and the player would have to fill in the remainder of the letters.

An example might look like this: c _ _ e. But there are too many solutions to this kind of a spelling game. Below, you can see a few, but there are probably hundreds.

  • cave
  • cafe
  • cove
  • come
  • cite

Subsequently, it occurred to me that if I could record my voice speaking a hard-to-spell word, then have the player attempt to spell it in a text box, I could evaluate the spelling, and pronounce it correct or not correct. That's what I did.

Development

To record my voice, I used a built-in Windows 10 app called Voice Recorder. Using a free tool, I converted the native Voice Recorder format, m4a, to mp3 format. Since there is little that can't be done with JavaScript, I figured that there must be a way to play audio files. I relied on this tutorial from w3schools.

Challenges

Ideally, a developer would want to place all scriping in a single js file, and source that file from somewhere within the html file. I found, however, that the JavaScript player would not work unless the script tag was internal to the html file.

If the player didn't have to worry about uppercase/lowercase, that would make the game more enjoyable. But the console would report errors on JavaScript's toLowerCase() method. My only choice was to prompt the user to use lower case only.
